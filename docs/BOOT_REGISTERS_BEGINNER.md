# Boot 阶段寄存器小白说明

这份文档不是手册风格，而是专门配合 `stage1.asm` 和 `stage2.asm` 看的。

目标只有一个：

> 让你知道这些寄存器在当前启动代码里分别是干什么的。

---

## 1. 通用寄存器

### `AX`

- 16 位通用寄存器
- 很多 BIOS 中断都爱用它传功能号和返回值

在这个项目里常见用途：

- `xor ax, ax`
  作用：把 `AX` 清零
- `mov ss, ax`
  作用：把栈段寄存器设成 0

### `AH` 和 `AL`

`AX` 可以拆成两个 8 位部分：

- `AH`：高 8 位
- `AL`：低 8 位

比如：

- `mov ah, 0x02`
  意思：给 BIOS `int 13h` 传“读扇区”功能号
- `mov al, 0x04`
  意思：要读 4 个扇区

### `BX`

- 16 位通用寄存器
- 常用来当基址或偏移

在这里：

- `mov bx, STAGE2_OFFSET`
  表示：把 `stage2` 读到 `ES:BX` 指向的位置

### `BH` 和 `BL`

`BX` 也能拆成：

- `BH`：高 8 位
- `BL`：低 8 位

在这里：

- `BL` 常拿来保存一个字节数据
- BIOS `int 10h` 里 `BL` 也会被当成颜色属性

### `CX`

- 16 位通用寄存器
- 常用于计数、循环，或者被 BIOS 中断拿来传参数

### `CH` 和 `CL`

`CX` 的高低 8 位：

- `CH`
- `CL`

在这里：

- `CH` 用来给 BIOS `int 13h` 传柱面号
- `CL` 用来传扇区号

### `DX`

- 16 位通用寄存器
- 在底层 I/O 里非常常见

在这里最重要的用途：

- `DX` 经常装“端口号”
- 比如串口端口 `0x3f8`
- 比如 QEMU debug-exit 端口 `0x00f4`

### `DH` 和 `DL`

`DX` 的高低 8 位：

- `DH`
- `DL`

在这里：

- `DL` 最重要
  BIOS 进入 boot sector 时，会把“从哪块盘启动”放在 `DL`
- `DH` 在 `int 13h` 读盘时表示磁头号

## 2. 指针/索引寄存器

### `SP`

- 16 位栈指针
- 和 `SS` 一起决定 16 位实模式下的栈位置

比如：

- `mov sp, 0x7a00`

意思是：

- 栈顶偏移设成 `0x7a00`

### `ESP`

- 32 位栈指针
- 进入保护模式后用它代替 `SP`

比如：

- `mov esp, PM_STACK_TOP`

表示：

- 32 位阶段重新换一个更合适的栈顶

### `SI`

- 16 位源索引寄存器
- 常被用作“字符串指针”

在这里你最常看到：

- `mov si, message`
- `lodsb`

意思是：

- `SI` 指向字符串
- `lodsb` 从 `DS:SI` 取一个字节到 `AL`
- 然后 `SI` 自动加 1

### `DI`

- 16 位目标索引寄存器
- 常当目的地址指针

在这里：

- `DI` 用来指向 `memory_map` 缓存区

### `ESI`

- 32 位版的 `SI`

进入保护模式后：

- `ESI` 用来指向 32 位字符串

### `EDI`

- 32 位版的 `DI`

进入保护模式后：

- `EDI` 用来指向 VGA 显存地址

## 3. 段寄存器

### `CS`

- 代码段寄存器
- CPU 当前从哪段代码里取指令，主要由它决定

你平时很少直接改它，通常通过：

- `jmp segment:offset`

这种“远跳转”来间接修改。

### `DS`

- 数据段寄存器
- 默认数据读取很多时候都从 `DS` 这个段里来

比如：

- `mov ds, ax`

就是在设置后面数据访问默认落在哪个段。

### `ES`

- 额外数据段寄存器

在这里常见用途：

- BIOS `int 13h` 读盘时，目标地址用 `ES:BX`

### `SS`

- 栈段寄存器

和：

- `SP`
- `ESP`

一起决定栈的位置。

### `FS` / `GS`

- 也是段寄存器
- 在当前这套代码里，进入保护模式后统一把它们也设成平坦数据段

这样做的目的只是：

- 保证所有段寄存器都处于一致、可预测的状态

---

## 4. 控制寄存器

### `CR0`

这是 CPU 的控制寄存器，不是普通通用寄存器。

在这里最关键的位是：

- `PE`，也就是 bit0

当你做：

- `mov eax, cr0`
- `or eax, 0x1`
- `mov cr0, eax`

意思就是：

> 把保护模式开关打开

所以 `CR0` 是“进入保护模式”的核心寄存器之一。

---

## 5. 为什么有时候用 16 位寄存器，有时候用 32 位寄存器

因为当前代码跨了两个阶段：

### 实模式阶段

你会看到：

- `AX`
- `BX`
- `CX`
- `DX`
- `SI`
- `DI`
- `SP`

因为这时候代码是：

- `bits 16`

### 保护模式阶段

你会看到：

- `EAX`
- `EBX`
- `ECX`
- `EDX`
- `ESI`
- `EDI`
- `ESP`

因为这时候代码已经切到：

- `bits 32`

所以不是“写法不统一”，而是 CPU 运行模式变了。

---

## 6. 你现在最该先记住的寄存器

如果你是小白，不要试图一次记住全部。

先记住这几个最重要的：

- `AX`
  通用寄存器，经常被 BIOS 中断拿来传功能号
- `AH` / `AL`
  `AX` 的高低 8 位
- `DX`
  经常装 I/O 端口号
- `DL`
  BIOS 传进来的启动盘号
- `SI`
  字符串指针
- `DI`
  目标缓冲区指针
- `DS`
  数据段
- `ES`
  额外数据段
- `SS`
  栈段
- `SP` / `ESP`
  栈指针
- `CR0`
  保护模式开关所在控制寄存器

把这些先看懂，再看 bootloader 代码就不会全是黑盒了。
