# 页表 + Long Mode 小白说明

这份文档讲的是下一轮为什么要做：

- 页表
- long mode

并且解释为什么它们必须按现在这个顺序来。

---

## 1. 为什么进入保护模式后还不能直接叫“64 位内核”

因为 32 位保护模式和 64 位 long mode 是两回事。

你现在已经做到的是：

- CPU 不再停留在 16 位实模式
- 代码能以 32 位方式运行

但这还不是最终的 `x86_64` 目标。

要真正进入 64 位，还必须额外满足：

1. 开启 `PAE`
2. 准备页表
3. 设置 `EFER.LME`
4. 打开分页
5. 跳进 64 位代码段

所以保护模式只是中间站，不是终点。

---

## 2. 为什么先做页表

进入 long mode 不是只改一个寄存器。

64 位模式要求分页机制参与进来，也就是说：

> 你必须先有页表，CPU 才知道 long mode 下地址怎么翻译。

如果没有页表，后面一旦打开分页：

- CPU 就不知道该怎么取指令
- 很容易直接黑屏或三重故障

所以顺序上一定是：

1. 先准备页表
2. 再开 long mode

---

## 3. 为什么这里先只映射低 2 MiB

现在我们不是在做完整内存管理，只是在做：

- 能成功进入 long mode 的最小页表

所以最简单的方案就是：

- 恒等映射低 2 MiB

什么叫恒等映射：

- 物理地址是多少，虚拟地址也是多少

比如：

- `0x8000` 还是映射到 `0x8000`
- `0xb8000` 还是映射到 `0xb8000`

为什么这样最合适：

- 当前 `stage2` 在低地址
- VGA 文本缓冲区 `0xb8000` 也在低地址
- 栈和页表也都在低地址

所以只映射前 2 MiB 就够当前实验用了。

但是这里有一个非常容易踩坑的地方：

- “低地址”不等于“这整段都是普通 RAM”

传统 PC 里：

- `0xA0000 ~ 0xBFFFF` 常常是显存或设备保留区
- `0xC0000 ~ 0xFFFFF` 也常常不是给你随便当普通内存用的

所以你虽然做了“低 2 MiB 恒等映射”，但不代表你能把栈随便丢到这些地址。

这也是为什么现在代码里把 long mode 栈放在：

- `0x9e000`

而不是放在：

- `0xa1000`

原因很简单：

- `call` 会往栈里压返回地址
- `ret` 会从栈里弹返回地址
- 如果这块“栈”其实落在显存/设备洞里，返回地址就可能写坏或读坏
- 结果通常就是 page fault，接着 double fault，最后 triple fault

对小白来说，你只要先记住一句就够：

> 栈一定要放在“确认是 RAM 的地址”里，不能只看它“数字不大”就拿来用。

---

## 4. 为什么要开 PAE

因为 long mode 的页表机制是建立在 PAE 这套扩展上的。

所以在进入 long mode 前：

- `CR4.PAE` 必须先打开

你可以把它理解成：

> “我要开始使用 long mode 所需要的页表格式了”

---

## 5. 为什么还要改 EFER

`EFER` 是扩展功能寄存器。

里面有个关键位：

- `LME`，Long Mode Enable

这一步的作用不是“立刻进入 64 位”，而是：

- 先告诉 CPU：我想要 long mode

真正进入还要等：

- 分页也打开
- 再跳到 64 位代码段

所以它更像一个“授权开关”。

---

## 6. 为什么最后还要 far jump

因为即使：

- PAE 开了
- LME 开了
- 分页开了

CPU 也还需要：

- 切到一个合法的 64 位代码段

这就是为什么最后必须做：

- far jump 到 64 位代码选择子

没有这一步，你不会真正开始执行 64 位代码。

---

## 7. 这一轮完成后你真正得到了什么

如果这一步成功，你得到的不是“完整内核”，而是：

```text
BIOS
-> stage1
-> stage2
-> A20
-> E820
-> GDT
-> protected mode
-> page tables
-> long mode
```

这条链一旦打通，你就真正摸到 `x86_64` 内核启动门槛了。

---

## 8. 为什么现在还不急着加载真正的 C++ 内核

因为现在最重要的是：

- 先把 CPU 模式切换链站稳

如果你现在一边切 long mode，一边再引入：

- ELF 加载
- C++ freestanding 运行时
- 更复杂的内存布局

一旦黑屏，你很难判断是：

- 页表错了
- GDT 错了
- 内核入口错了
- 加载地址错了

所以现在先把：

- `long mode ok`

这个里程碑独立做出来，是最稳的。
